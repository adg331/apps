什么是 MCP？

Model Context Protocol 是连接 AI 模型与外部系统的开放标准，好比 AI 世界的 "USB-C 接口"。NetApp 把存储的查询与管理动作封装为一组组显式定义、参数受校验的工具（Tools），AI 不再"猜测"如何运维存储，而是调用结构化接口——可预测、可审计、有护栏。这是 NetApp 的一项整体战略，而非单点尝试。

两大核心：观察 + 操作

Harvest MCP · 观察

会话式可观测性

构建在 Harvest 开源监控管道之上，将 Prometheus / VictoriaMetrics 中的时序指标变成 AI 可查询的数据层。覆盖 ONTAP、StorageGRID、E-Series、Cisco 交换机——不用写 PromQL、不用翻仪表盘，直接问："基础设施现在健康吗？哪些卷延迟在上升？"

ONTAP MCP · 操作

会话式存储运维

轻量 Go 服务，将 ONTAP 管理动作映射为 REST API 工作流：卷生命周期、NFS/CIFS 访问控制、快照与 SnapMirror 数据保护、QoS、SAN/NVMe、多集群管理。AI 获得"双手"，但带着护栏——参数校验、只读模式、危险操作标注。

系统架构

一个对话入口，四条能力路径：Harvest MCP 做观察，ONTAP MCP 做操作，DataOps Toolkit MCP ×4 做自助服务；知识检索则分成两条独立路径：AIDE MCP 连接 AIDE 企业 RAG，Workload Factory GenAI KB MCP 连接 AWS 侧 Knowledge Base。MCP 只是 Agent 侧适配层，检索服务位于各自后端。点击图中组件查看详细说明。

关键设计要点

Harvest MCP

轻量、可加装

不引入新数据管道：已部署 Harvest + Prometheus/VictoriaMetrics 的环境，只需多跑一个容器即可让存储遥测"开口说话"。

Harvest MCP

统一指标模型

ONTAP、StorageGRID、E-Series、Cisco 用一致的开放指标模型，跨平台提问一次到位；Grafana 仪表盘依然可用，二者互补。

ONTAP MCP

带护栏的"双手"

每个工具标注 readOnly / idempotent / destructive 提示；--read-only 模式仅暴露只读工具；集中凭证，调用不带密码。

DataOps MCP

本地 stdio · 零门槛

uvx 一条命令即可拉起，凭证与配置留在本地；面向终端用户自助服务，覆盖本地 ONTAP 与 Azure / Google 云、K8s。

知识 MCP

两条 RAG 路径，边界清晰

AIDE MCP 面向 AIDE 企业数据检索；Workload Factory GenAI KB MCP 面向 AWS 侧知识库。两者都把受认证的检索能力交给 Agent，但数据位置、身份体系和治理边界取决于各自后端。

观察 操作 知识

同一对话，全栈协同

一个 mcp.json 可同时接入远程 HTTP 与本地 stdio 服务器，AI 在同一对话流内查指标、做变更、检索知识。

知识检索 MCP ×2　企业知识 RAG

🧠 AIDE MCP（AI Data Engine）

netapp_data_engine_search

面向 AIDE 平台的数据检索入口。MCP Server 将 AIDE RAG Retrieval Endpoint 封装为标准 MCP Tool， AI Agent 可通过语义搜索访问 AIDE 已建立的数据集合、向量索引和企业文档知识。 MCP 不是检索引擎本身，也不是必须组件；自研应用可以直接调用 AIDE RAG API。

适合：AFX + AIDE、本地 AI 数据平台、企业私有数据 RAG
"查找 ONTAP 故障处理手册中关于 SnapMirror 回切的标准流程"

📚 Workload Factory GenAI KB MCP

knowledge base search

面向 NetApp Workload Factory GenAI Knowledge Base 的 Agent 接入层。 将已经在 AWS 环境中构建完成的 GenAI Knowledge Base 暴露给 MCP Client， 通常结合 Amazon Bedrock、FSx for ONTAP 和 Workload Factory GenAI 工作流使用。 MCP Server 可在本地电脑或 Agent 服务器侧运行；知识库与 RAG 服务在 AWS 侧。它用于知识查询，不创建、不扩容、也不管理 FSx for ONTAP。

适合：AWS + FSx for ONTAP + Amazon Bedrock RAG 场景
"从云上运维知识库查找类似故障处理记录"

快速部署

两个 MCP Server 都以单容器交付，几分钟即可上线。可以只部署其一，也可以组合部署形成"观察 + 操作"闭环。

Harvest MCP　部署（观察）

前提：已有 Harvest 将指标导出到 Prometheus 或 VictoriaMetrics · Docker 环境 · MCP Server 到时序库的网络连通。如果 Harvest 已经在运行，启用 MCP 只是一个轻量的加装步骤。

1

运行 Harvest MCP Server 容器

docker run -d \
  --name harvest-mcp-server \
  -p 8082:8082 \
  --env HARVEST_TSDB_URL=http://your-prometheus:9090 \
  ghcr.io/netapp/harvest-mcp:latest \
  start --http --port 8082 --host 0.0.0.0
# 启动 MCP API 服务 → 直连你的时序库 → 遥测数据即可被 AI 发现和查询
# 也支持本地 stdio 模式，适合桌面端直连
2

连接 AI 助手并提问

# 任何 MCP 兼容客户端指向该端点后，直接问：
"How's my infrastructure doing right now?"
"Which ONTAP volumes are approaching capacity limits?"
"What are the top five volumes by latency for the last 30 minutes?"

不用新建仪表盘、不用写查询语句——遥测数据直接变成对话。

ONTAP MCP　部署（操作）

前提：ONTAP 集群及管理员凭证 · Docker 环境 · MCP Server 到集群的网络连通。

1

启动 ONTAP MCP Server 容器

docker run -d \
  --name ontap-mcp-server \
  -p 8083:8083 \
  -v /path/to/your/ontap.yaml:/opt/mcp/ontap.yaml \
  ghcr.io/netapp/ontap-mcp:latest \
  start --port 8083 --host 0.0.0.0   # 可加 --read-only 只读模式
# ontap.yaml — 集中注册集群凭证（一次注册，多集群纳管）
Pollers:
  cluster1:
    addr: 10.0.0.1
    username: admin
    password: ********
    use_insecure_tls: true
2

开始对话

# 在 Claude Desktop / Copilot 聊天窗口中直接输入：
"create a volume of 5 gb with name demo_vol01
 in svm demo_svm01 and aggregate agg_data_tme_1"

AI 负责编排，ONTAP MCP 确保操作安全、可预测地执行。

DataOps Toolkit MCP ×4　部署（自助服务 · 本地 stdio）

前提：Python ≥ 3.8 · uv 或 pip。本地 stdio 服务器由 MCP 客户端自动拉起，凭证与配置保留在用户本机。ANF / GCNV / K8s 版部署方式类似，见各自官方文档。

1

生成 DataOps Toolkit 配置（首次）

# 交互式创建配置文件（集群地址、SVM、凭证等）
uvx --from netapp-dataops-traditional netapp_dataops_cli.py config
2

在 mcp.json 中注册（无需预装，uvx 按需拉起）

{
  "mcpServers": {
    "netapp_dataops_ontap_mcp": {
      "type": "stdio",
      "command": "uvx",
      "args": ["--from", "netapp-dataops-traditional", "netapp_dataops_ontap_mcp.py"]
    }
  }
}

之后直接说："把 prod_dataset 克隆一份挂到 /mnt 给训练用。"

AIDE MCP　Agent 侧适配器（连接 AIDE RAG）

以下配置仅适用于 AIDE MCP。前提：Python ≥ 3.10 · uv · AIDE Retrieval/RAG Endpoint 与 OAuth2 应用信息（PKCE 或设备码流程）。MCP Server 在 Agent 侧运行，真正的检索由 AIDE 后端完成。

1

创建 ~/.netapp 认证配置

{
  "rag_search_api_endpoint_url": "https://your-aide/api",
  "auth_flow": "pkce",
  "token_request_endpoint_url": "https://login.microsoftonline.com/<tenant>/oauth2/v2.0/authorize",
  "token_exchange_endpoint_url": "https://login.microsoftonline.com/<tenant>/oauth2/v2.0/token",
  "token_request_params": { "client_id": "your_client_id", "redirect_uri": "http://localhost:8888", "use_pkce": true }
}
# chmod 600 ~/.netapp（首次工具调用时触发浏览器认证）
2

在 mcp.json 中注册 AIDE MCP

{
  "servers": {
    "netapp-aide-mcp": {
      "type": "stdio",
      "command": "uvx",
      "args": ["--from", "netapp-aide-mcp", "server"]
    }
  }
}

Workload Factory GenAI KB MCP　Agent 侧适配器（连接 AWS 知识库）

该 MCP Server 可由桌面客户端或 Agent 服务器以 stdio 方式拉起，再通过 HTTPS/API 访问 AWS 侧 Workload Factory GenAI Knowledge Base。需要相应的 BlueXP/Workload Factory 服务账号与可访问的已发布知识库。

1

先在 AWS 侧准备知识库

在 Workload Factory GenAI 中创建并发布 Knowledge Base，配置其数据源、模型与访问策略。FSx for ONTAP 可以承载或提供数据，但 KB MCP 本身不是 FSx 管理工具。

2

在 Agent 环境注册 KB MCP

从 NetApp MCP 目录获取当前版本的安装与配置方法，并按其说明提供 Account ID、Client ID 与 Client Secret。不要沿用上方 AIDE 的 ~/.netapp OAuth 配置。

🔗 组合配置：一个 mcp.json 接入整个家族

{
  "servers": {
    "harvest-mcp": { "type": "http",  "url": "http://your-server-ip:8082" },
    "ontap-mcp":   { "type": "http",  "url": "http://your-server-ip:8083" },
    "dataops-ontap": { "type": "stdio", "command": "uvx", "args": ["--from","netapp-dataops-traditional","netapp_dataops_ontap_mcp.py"] },
    "netapp-aide-mcp": { "type": "stdio", "command": "uvx", "args": ["--from","netapp-aide-mcp","server"] }
  }
}

远程 HTTP 与 Agent 侧 stdio 服务器可共存于一个配置。注意：配置片段只展示了 AIDE MCP；Workload Factory GenAI KB MCP 需按其独立认证与配置说明注册。Agent 可以同时使用两者，但它们连接的是不同 RAG 后端，并不共享知识库或权限。
