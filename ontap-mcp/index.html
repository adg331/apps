什么是 MCP？

Model Context Protocol 是连接 AI 模型与外部系统的开放标准，好比 AI 世界的 "USB-C 接口"。ONTAP MCP Server 把 ONTAP 管理操作封装为一组显式定义、参数受校验的工具（Tools），AI 不再"猜测"如何管理存储，而是调用结构化接口——可预测、可审计、有护栏。

关键设计要点

🔁

Streamable HTTP

支持多客户端并发连接；--stateless 模式可兼容剥离会话头的代理环境。

🔑

集中凭证

集群凭证在 ontap.yaml 中一次注册，工具调用无需携带凭证——更安全、更简单。

🏷️

工具提示元数据

每个工具标注 readOnlyHint / idempotentHint / destructiveHint，--read-only 模式仅暴露只读工具。

MCP 工具能力

每个工具对应一个定义良好的 ONTAP REST API 工作流，覆盖 NAS、SAN、数据保护与多集群管理。点击分类展开工具列表。

青色 = 只读/查询类　橙色 = 写入/变更类（带 destructiveHint 护栏）

三步快速部署

前提条件：ONTAP 集群及管理员凭证 · Docker 运行环境 · MCP Server 到集群的网络连通。

1

启动 ONTAP MCP Server 容器

docker run -d \
  --name ontap-mcp-server \
  -p 8083:8083 \
  -v /path/to/your/ontap.yaml:/opt/mcp/ontap.yaml \
  ghcr.io/netapp/ontap-mcp:latest \
  start --port 8083 --host 0.0.0.0   # 可加 --read-only 只读模式
# ontap.yaml — 集中注册集群凭证
Pollers:
  cluster1:
    addr: 10.0.0.1
    username: admin
    password: ********
    use_insecure_tls: true
2

在 AI 客户端配置 mcp.json

{
  "servers": {
    "ontap-mcp": {
      "type": "http",
      "url": "http://your-server-ip:8083"
    }
  }
}
3

开始对话

# 在 Claude Desktop / Copilot 聊天窗口中直接输入：
"create a volume of 5 gb with name demo_vol01
 in svm demo_svm01 and aggregate agg_data_tme_1"

AI 负责编排，ONTAP MCP 确保操作安全、可预测地执行。
